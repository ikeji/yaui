# yaui Protocol

3 つのランタイム（`yaui-tui` / `yaui-gtk` / `yaui-web`）すべてが共通で扱う、スクリプトとの通信仕様。

## 流れ

```
              ┌─────────┐
   stdin ←──  │ runtime │  ──→ stdout
   (events)   │         │      (UI trees)
              └─────────┘
                  │
              ┌───┴───┐
              │script │
              └───────┘
```

- **スクリプト → ランタイム（stdout）**: JSON の UI ツリー。ランタイムはブレース深度を追って 1 オブジェクトずつ取り出すので、複数行にまたがって構わないし、オブジェクトの間に空白や改行があっても良い。`{` / `[` で始まらないバイトはスキップされるため、テスト目的でイベントを `echo` し返してもパースを壊さない。
- **ランタイム → スクリプト（stdin）**: 1 行 1 オブジェクトの JSON イベント。

## UI ツリー

ルートは `type` を持つ JSON オブジェクト。通常は `dialog`（または `sdi`）が最上位だが、それ以外の単体ウィジェットでも動く。

### 共通フィールド

| フィールド | 型      | 説明                                                                         |
|------------|---------|------------------------------------------------------------------------------|
| `type`     | string  | 必須。ウィジェット種別（下記参照）                                           |
| `id`       | string  | 任意。イベントペイロードに同梱され、TUI では `id` キーで状態が保持される     |

## ウィジェット一覧

### `dialog` / `sdi`（ルートコンテナ）

| フィールド | 型     | 説明                            |
|------------|--------|---------------------------------|
| `title`    | string | 任意。ウィンドウタイトル        |
| `content` / `children` | array \| object | 中身。配列・単体オブジェクトどちらでも可 |

```json
{"type": "dialog", "title": "Hello",
 "content": [{"type": "label", "text": "world"}]}
```

### `vbox` / `wbox`（縦スタック）/ `hbox`（横スタック）

| フィールド            | 型              |
|-----------------------|-----------------|
| `children` / `content` | array           |
| `spacing`             | int（GTK のみ） |

`wbox` は `vbox` の別名。

### `label`

| フィールド | 型      | 説明                                  |
|------------|---------|---------------------------------------|
| `text`     | string  | 表示文字列。`\n` で複数行可           |
| `bold`     | bool    | 任意。太字表示                        |

### `button`

| フィールド | 型     |
|------------|--------|
| `text`     | string |

→ 押下で `click` イベント。

### `checkbox`

| フィールド | 型     | 説明      |
|------------|--------|-----------|
| `text`     | string | ラベル    |
| `checked`  | bool   | 初期状態  |

→ トグルで `change` イベント（`value: <bool>`）。

### `textbox`（一行入力）

| フィールド | 型     | 説明                              |
|------------|--------|-----------------------------------|
| `value`    | string | 初期値。再エミット時に上書きされる |
| `width`    | int    | 横幅ヒント（文字数）              |
| `controlled` | bool | TUI で `value` を毎回適用させる   |

→ 入力で `change`、Enter で `submit`（どちらも `value: <string>`）。

### `textarea`（複数行入力）

| フィールド | 型     | 説明           |
|------------|--------|----------------|
| `value`    | string | 初期値（改行可）|
| `width`    | int    | 桁ヒント       |
| `height`   | int    | 行数ヒント     |
| `controlled` | bool | 同上           |

→ 入力で `change`（submit はなし）。

### `list`

| フィールド | 型     | 説明                                                        |
|------------|--------|-------------------------------------------------------------|
| `items`    | array  | 文字列か `{text\|label\|value}` を持つオブジェクト         |
| `selected` | int    | 初期選択インデックス                                        |
| `height`   | int    | 行数ヒント                                                  |

→ 選択変更で `change`（`value: <int>`, `item: <string\|object>`）、行アクティベート（TUI: Enter / GTK: ダブルクリック / Web: ダブルクリック）で `select`（同じペイロード）。

### `progressbar`

| フィールド  | 型     | 説明                       |
|-------------|--------|----------------------------|
| `value`     | number | 現在値                     |
| `max`       | number | 既定 100                   |
| `width`     | int    | 桁ヒント（TUI/GTK）        |
| `show_text` | bool   | パーセント表示             |

→ 非インタラクティブ。

## イベント

すべて 1 行 1 オブジェクトで stdin に届く。

| `type`   | 発火タイミング                                         | 追加フィールド                |
|----------|--------------------------------------------------------|-------------------------------|
| `click`  | ボタン押下                                             | `id`（設定されていれば）      |
| `change` | 入力値・選択の変更                                     | `id`, `value`（場合により `item`） |
| `submit` | textbox で Enter                                        | `id`, `value`                 |
| `select` | list の行をアクティベート                               | `id`, `value`, `item`          |
| `close`  | ユーザーがダイアログを閉じる / Esc                      | （追加なし）                  |

例:

```json
{"type": "click", "id": "ok"}
{"type": "change", "id": "name", "value": "Bob"}
{"type": "select", "id": "fruit", "value": 1, "item": "banana"}
{"type": "close"}
```

## ライフサイクル

1. ランタイムがスクリプトを起動
2. スクリプトが UI ツリーを stdout に出力
3. ランタイムが描画
4. ユーザーが操作 → ランタイムがイベント JSON を 1 行で stdin に書く
5. スクリプトが受け取って状態を更新し、新しい UI ツリーを stdout に出して 3. に戻る
6. `close` を受けたら、または stdin が EOF になったら、スクリプトは終了

## 状態保持の挙動

ランタイムは UI ツリーを受信するたびに「全再構築」する。ただしユーザー操作中の状態を失わない工夫が入る。

- **TUI**: ウィジェット状態（textbox の文字列・カーソル、checkbox、list 選択、textarea）は `id` をキーに保持。新しいツリーの `value` は無視される。明示的に上書きしたい場合は `controlled: true` を付ける。
- **GTK / Web**: 毎回作り直すため、再描画の前に保持していた値は失われる。よって、textbox/textarea のキー入力ごとに再エミットすると、カーソルが先頭に飛ぶなどの問題が起きる。デモスクリプトでは「textbox/textarea の `change` は変数を更新するだけで再描画しない、その他のイベント（ボタン押下など）に応じて再エミットする」というパターンを採用している。

## 環境変数

ランタイムはスクリプトの環境に `YAUI=tui|gtk|web` を設定する。スクリプトはこれを見て描画先を分岐できる。

## ストリームの解析

JSON パーサはバイト単位で `{` / `[` の深さを追い、文字列リテラル中（`"..."`）とエスケープを正しく扱う。深さ 0 に戻った時点でその範囲をデコードして 1 オブジェクトとして produce する。途中で `\n` が来ても、空白文字も、すべて受け流す。
